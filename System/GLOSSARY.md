## 基礎概念

### NAV（淨資產價值）

你的投資帳戶總共值多少錢。

```
NAV = 帳戶裡的現金 + 所有股票的市值總和
```

例如：帳戶有 200 萬現金，持有台積電市值 500 萬、鴻海市值 300 萬：

```
NAV = 200萬 + 500萬 + 300萬 = 1,000萬
```

### 持倉（Position）

你目前持有某支股票的狀態。

```
市值 = 持有數量 × 目前股價
未實現損益 = (目前股價 - 買入均價) × 持有數量
```

例如：買了 1,000 股台積電，當時均價 550 元，現在漲到 600 元：

```
市值 = 1,000 × 600 = 60萬
未實現損益 = (600 - 550) × 1,000 = +5萬（帳面賺 5 萬，但還沒賣）
```

### 權重（Weight）

某支股票在你投組中佔多少比例。

```
某股票的權重 = 該股票市值 / NAV
```

例如：NAV 1,000 萬，台積電市值 500 萬 → 權重 = 500 / 1000 = **50%**。

這個系統中，策略回傳的就是「目標權重」— 告訴系統每支股票應該佔多少比例。

### 槓桿（Leverage）

你借了多少錢來投資。

```
槓桿 = 所有持倉的市值絕對值總和 / NAV
```

- 槓桿 = 1.0 → 沒有借錢，用自己的錢投資
- 槓桿 = 1.5 → 借了 50% 的錢來多買
- 槓桿 = 0.5 → 只投了一半的錢，另一半放現金

### 做多 vs 做空

- **做多（Long）**：先買進，等漲了再賣掉賺差價。「看漲」。
- **做空（Short）**：先借股票來賣，等跌了再買回來還。「看跌」。

做空比較複雜，需要付借券費，而且虧損理論上無上限（股價可以一直漲）。

## 報酬指標

### 總報酬（Total Return）

整段時間賺了多少。

```
總報酬 = (期末 NAV / 期初資金) - 1
```

例如：投入 1,000 萬，最後變成 1,500 萬：

```
總報酬 = (1500 / 1000) - 1 = 50%
```

### CAGR（年化複合報酬率）

每年平均賺多少，考慮複利效果。

```
年數 = 交易天數 / 252     （一年約 252 個交易日）
CAGR = (1 + 總報酬) ^ (1 / 年數) - 1
```

為什麼需要 CAGR？因為「4 年賺 60%」和「1 年賺 60%」差很多。

```
4 年賺 60% → CAGR = (1.60) ^ (1/4) - 1 ≈ 12.5% 每年
1 年賺 60% → CAGR = 60%
```

> **系統中的門檻：CAGR ≥ 8%**。因為台股大盤長期年化約 8-10%，你的策略至少要打敗「什麼都不做、買大盤就好」。

### Sharpe Ratio（夏普比率）

**最重要的指標之一。** 衡量「每承受一單位風險，你賺到多少報酬」。

```
每日報酬率 = 每天 NAV 變化的百分比
Sharpe = (每日報酬率的平均值 / 每日報酬率的標準差) × √252
```

白話：你每天的報酬有多穩定？如果平均每天賺 0.1%，但有時候賺 5%、有時候虧 4%，那 Sharpe 就很低。如果平均每天賺 0.05%，但幾乎每天都在 0.03%-0.07% 之間，Sharpe 反而更高。

| Sharpe | 評價 |
|--------|------|
| < 0 | 虧錢 |
| 0 ~ 0.5 | 不值得冒這個風險 |
| 0.5 ~ 1.0 | 還行 |
| 1.0 ~ 2.0 | 很好 |
| > 2.0 | 非常好，但要小心是不是過擬合 |

> **系統中的門檻：Sharpe ≥ 0.7**

### Sortino Ratio（索提諾比率）

跟 Sharpe 很像，但只算**下跌**的風險。

```
下行波動率 = 只取虧損那些天的報酬率算標準差
Sortino = 年化平均報酬 / 下行波動率
```

為什麼有人偏好 Sortino？因為如果你今天賺了 5%，那是好事，不應該被當成「風險」。Sharpe 把大漲大跌都當風險，Sortino 只懲罰虧損。

### Calmar Ratio（卡瑪比率）

年化報酬除以最大回撤。

```
Calmar = CAGR / 最大回撤
```

白話：你每年賺的報酬，能不能覆蓋你「最慘那段時間」的虧損？

```
CAGR 12%、最大回撤 30% → Calmar = 0.4（賺的還不夠填那個坑）
CAGR 20%、最大回撤 15% → Calmar = 1.33（很不錯）
```

### Omega Ratio

所有賺錢的總和 / 所有虧錢的總和。

```
Omega = Σ max(每日報酬, 0) / Σ max(-每日報酬, 0)
```

Omega = 1 表示賺虧打平；> 1 表示賺的比虧的多。比 Sharpe 更直觀但用得較少。

## 風險指標

### 波動率（Volatility）

報酬率的「不穩定程度」。

```
波動率 = 每日報酬率的標準差 × √252    （年化）
```

標準差是統計學概念：衡量一組數字偏離平均值有多遠。如果你每天報酬在 -1% 到 +1% 之間跳，日波動率大約 1%，年化大約 16%。

| 年化波動率 | 白話 |
|-----------|------|
| 5-10% | 很穩定（類似債券） |
| 15-25% | 正常股票水平 |
| 30%+ | 很波動（新興市場或小型股） |

### 最大回撤（Max Drawdown, MDD）

從最高點跌到最低谷，你最多虧了多少。

```
每一天的回撤 = (當天 NAV - 歷史最高 NAV) / 歷史最高 NAV
最大回撤 = 所有回撤中最大的那個
```

```
比如你的 NAV 曾經是：
100萬 → 120萬 → 110萬 → 90萬 → 130萬

最高點 120萬 → 最低點 90萬
最大回撤 = (120 - 90) / 120 = 25%
```

**為什麼重要？** 因為最大回撤是你「真的會感受到的痛苦」。年化報酬 15% 聽起來很好，但如果中間有一段虧了 50%，大多數人早就嚇跑了。

> **系統中的門檻：MDD ≤ 40%**

### 回撤持續天數

你待在「虧損狀態」多久。

```
回撤持續天數 = 從 NAV 跌破前高到恢復前高所經過的交易日數
```

跌 20% 然後 3 個月回來，跟跌 20% 然後 2 年才回來，體驗完全不同。

### VaR（風險值）— 95%

在正常情況下，一天最多虧多少。

```
VaR(95%) = 把所有每日報酬率從小排到大，取第 5% 的那個值
```

例如：你有 1,000 天的資料，VaR(95%) 就是第 50 差的那天的報酬率。

```
VaR(95%) = -2% → 意思是「有 95% 的天數，你當日虧損不會超過 2%」
```

### CVaR（條件風險值）— 95%

最慘那 5% 的天數，平均虧多少。

```
CVaR(95%) = 所有低於 VaR 的那些日子的平均報酬率
```

VaR 告訴你「最壞情況的門檻」，CVaR 告訴你「比那個門檻更慘的時候平均有多慘」。

```
VaR(95%) = -2%
CVaR(95%) = -3.5% → 最慘那 5% 的日子平均虧 3.5%
```

> **系統中的門檻：CVaR(95%) ≥ -5%**（一天最多虧 5%）

### 市場相關性

你的策略跟大盤的連動程度。

```
市場相關性 = 你的策略每日報酬 vs 大盤每日報酬 的相關係數
```

- 相關性 = 1.0 → 你的策略就是大盤的翻版
- 相關性 = 0.0 → 完全獨立，大盤漲跌跟你無關
- 相關性 = -1.0 → 大盤漲你就跌（反向）

> **系統中的門檻：|相關性| ≤ 0.80**。如果相關性太高，你不如直接買 ETF，何必自己做策略。

## 交易成本

### 手續費（Commission）

每次交易要付給券商的費用。

```
手續費 = 成交金額 × 手續費率
台股手續費率 = 0.1425%（買賣各收一次）
最低手續費 = 20 元（整張）/ 1 元（零股）
```

例如：買 100 萬的股票 → 手續費 = 100萬 × 0.001425 = 1,425 元。

### 證交稅（Trading Tax）

賣出股票時繳給政府的稅。

```
證交稅 = 賣出金額 × 稅率
台股稅率 = 0.3%（只有賣出時收）
```

例如：賣 100 萬的股票 → 證交稅 = 100萬 × 0.003 = 3,000 元。

### 滑價（Slippage）

你下單的時候，實際成交價跟你預期的價格之間的差距。

為什麼會有滑價？因為你在買的時候，賣方可能把價格抬高一點；你在賣的時候，買方可能壓低價格。你買的量越大，影響越明顯。

**固定滑價模型：**

```
滑價 = 股價 × 滑價基點 / 10000
買入成交價 = 市價 + 滑價
賣出成交價 = 市價 - 滑價

例如：5 個基點滑價，股價 100 元
滑價 = 100 × 5 / 10000 = 0.05 元
→ 買入成交價 100.05 元
```

**根號衝擊模型（更接近現實）：**

```
市場參與率 = 你的下單量 / 當天成交量
衝擊基點 = 基礎滑價 + 衝擊係數 × √(市場參與率)
滑價 = 股價 × 衝擊基點 / 10000
```

白話：你如果買了當天成交量的 1%，影響很小。但如果你買了 10%，你自己就把價格推高了。這個公式用根號來模擬這個效果。

> **為什麼成本很重要？** 台股研究的結論是：換手率超過 10 倍/年的策略全部虧損。手續費 + 稅 + 滑價加起來，每次交易大約吃掉 0.5-1% 的利潤。如果你每個月全部換股一次，一年成本就是 6-12%，幾乎把報酬吃光。

### 漲跌停

台股規定：一支股票一天最多漲或跌 10%。

```
如果今天已經漲停（+10%）→ 買單被拒絕（買不到）
如果今天已經跌停（-10%）→ 賣單被拒絕（賣不掉）
```

這是真實市場的限制，回測必須模擬，否則結果會過於樂觀。

### 整張 vs 零股

台股：1 張 = 1,000 股。

```
如果股價 500 元，買 1 張 = 500 × 1000 = 50萬
零股：可以買 1-999 股，但流動性差、成交價較差
零股額外滑價 = +10 基點
```

## 因子研究

### 因子（Factor）

一個數字，用來「預測」股票未來的報酬。

例如 revenue_acceleration（營收加速度）：

```
revenue_acceleration = 近 3 個月平均營收 / 近 12 個月平均營收
```

- 比率 > 1 → 最近幾個月的營收比過去一年的平均高，公司在加速成長
- 比率 < 1 → 公司營收在減速

### IC（資訊係數）

衡量因子**預測力**的數字。

```
每一天：
  1. 算出所有股票的因子值（例如每支股票的 revenue_acceleration）
  2. 算出每支股票未來 20 天的報酬率
  3. 計算兩者的排名相關性
  IC = 因子排名 和 未來報酬排名 的相關係數
```

- IC = +0.10 → 因子值高的股票，未來報酬確實比較好（弱正相關）
- IC = 0 → 因子沒有預測能力
- IC = -0.10 → 因子值高的股票，未來報酬反而比較差

> 量化投資中，IC = 0.05 就已經「有用」了。不要覺得 0.05 很小 — 在高度隨機的股市中，能穩定產生 5% 的相關性已經很厲害。

### ICIR（資訊係數資訊比率）

IC 的「穩定度」。

```
ICIR = IC 的平均值 / IC 的標準差
```

為什麼重要？有些因子平均 IC 很高，但忽正忽負不穩定；有些因子平均 IC 不高，但幾乎每個月都是正的。ICIR 獎勵穩定性。

```
因子 A：IC 平均 0.10，標準差 0.20 → ICIR = 0.50
因子 B：IC 平均 0.05，標準差 0.02 → ICIR = 2.50

因子 B 的預測力雖然弱，但非常穩定 → ICIR 反而更高
```

> **revenue_acceleration 的實證結果：ICIR(60日) = 0.582**，這在因子研究中算很好。

### 前視偏差（Look-Ahead Bias）

回測時最隱蔽的錯誤：不小心用了「當時還不知道」的資訊。

```
錯誤：3 月 1 日的策略用了 2 月的營收數據
事實：2 月營收最晚 3 月 10 日才公布，3 月 1 日根本不知道

正確做法：3 月 1 日只能用 1 月（或更早）的營收數據
本系統強制加 40 天延遲，確保不會犯這個錯
```

> 本系統曾經因為沒加 40 天延遲，所有因子的 IC 被高估了 72%（0.188 → 0.674）。結果看起來「完全正常」，但全是假的。

### 前進報酬（Forward Return）

未來 N 天的報酬率。用來衡量因子是否真的能「預測」。

```
前進報酬(20天) = (20天後的股價 / 今天的股價) - 1
```

注意：前進報酬只在研究階段用來驗證因子，不會在實際交易中使用（因為你不知道未來的價格）。

## 投組建構

有了因子值，怎麼決定每支股票買多少？

### 縮尾處理（Winsorization）

去除極端值，避免少數異常公司主導整個投組。

```
把所有股票的因子值排序
低於第 1 百分位的 → 設定為第 1 百分位的值
高於第 99 百分位的 → 設定為第 99 百分位的值
```

例如：100 支股票的營收加速度大多在 0.8-1.5 之間，但有一支是 10.0（營收暴漲）。縮尾後那支被壓到 1.5，避免它主導投組。

### Z-score 標準化

讓不同因子的量綱可以比較。

```
Z-score = (某支股票的因子值 - 所有股票因子值的平均) / 標準差
```

例如：revenue_acceleration 的範圍是 0.5-2.0，ROE 的範圍是 5%-30%。直接比沒意義，標準化後都變成「距離平均值幾個標準差」，就可以放在一起比了。

### 中性化（Neutralization）

去除「共同因素」的影響，只留下個股的差異。

**市場中性化：**

```
每一天：
  中性化後的值 = 原始因子值 - 所有股票因子值的平均
```

為什麼？大盤漲的那個月，所有公司的營收都好看。市場中性化移除這個「大家都漲」的部分，只看「誰比平均好」。

### 等權配置（Equal Weight）

最簡單的配置方式：每支股票買一樣多。

```
權重 = 1 / 選入的股票數量

選了 15 支 → 每支 6.67%
```

### 訊號加權（Signal Weight）

因子值越強的股票，買越多。

```
某股票的權重 = 該股票的因子值 / 所有選入股票因子值的總和
```

### 風險平價（Risk Parity）

波動大的股票少買一點，波動小的股票多買一點，讓每支股票對投組的「風險貢獻」相等。

```
某股票的權重 = (1 / 該股票的波動率) / Σ(1 / 所有股票的波動率)
```

例如：台積電波動率 20%，聯發科波動率 40%。風險平價會讓台積電的權重是聯發科的 2 倍，因為它更穩定。

## 防過擬合

### 什麼是過擬合（Overfitting）？

策略在歷史數據上表現很好，但在未來會失效。

想像你要預測明天的天氣。如果你的模型是「記住過去 10 年每一天的天氣」，它在歷史數據上 100% 正確，但對明天完全沒用。這就是過擬合 — 你「記住了歷史」而不是「學到了規律」。

量化交易中最常見的過擬合：
- 試了 100 個策略，選最好的那個 → 可能只是運氣
- 針對特定時間段調整參數 → 換個時間段就失效
- 用太少的股票回測 → 選擇偏差

### Deflated Sharpe Ratio (DSR)

「你試了很多策略之後，選出的那個 Sharpe 還可信嗎？」

```
基本想法：
  如果你只試了 1 個策略，Sharpe 1.0 可能是真的
  如果你試了 1000 個策略，選出 Sharpe 最高的那個，
  就算所有策略都是隨機的，也會有一個「看起來不錯」
  
DSR 用數學修正這個偏差：
  考慮 (1) 試了多少個策略 (2) 有多少天的數據 (3) 報酬分佈的偏態和峰態
  計算出「這個 Sharpe 是真的」的機率
```

> **系統中的門檻：DSR ≥ 0.70**（70% 以上機率是真的）

### 前推分析（Walk-Forward Analysis）

模擬「用過去的資料做決策、看未來的結果」。

```
做法：
  第 1 折：用 2017-2018 訓練 → 測試 2019 的表現
  第 2 折：用 2018-2019 訓練 → 測試 2020 的表現
  第 3 折：用 2019-2020 訓練 → 測試 2021 的表現
  ...

前推一致性 = 樣本外表現為正的折數 / 總折數
```

> **系統中的門檻：≥ 60% 的折要是正的**。如果用過去訓練出來的策略，在「沒看過的未來」只有 30% 的時間賺錢，那這個策略不可靠。

### Bootstrap（自助重抽樣）

不假設報酬的分佈，直接從歷史數據中「抽籤」來看結果有多穩定。

```
做法：
  你有 1000 天的報酬率
  從這 1000 天中隨機抽出 1000 天（可以重複抽）
  用抽出的數據算一次 Sharpe
  重複這個過程 1000 次
  → 得到 1000 個 Sharpe 值的分佈

Bootstrap 信心 = 這 1000 次中 Sharpe > 0 的比例
```

> **系統中的門檻：P(Sharpe > 0) ≥ 80%**

### PBO（回測過擬合機率）

直接計算「這個策略是過擬合」的機率。

```
做法 (CSCV, Bailey et al. 2017)：
  1. 把歷史分成 S 段（比如 10 段）
  2. 用組合數學列出所有「取一半當訓練、一半當測試」的方式
  3. 每種方式：找出訓練集裡表現最好的策略，看它在測試集的排名
  4. PBO = 「最好的策略在測試集中排名反而很差」的比例
```

- PBO < 0.5 → 可能不是過擬合
- PBO > 0.5 → 很可能是過擬合

> **系統中的門檻：PBO ≤ 0.50**

### 排列檢定（Permutation Test）

「如果因子和報酬的關係是隨機的，也能得到這麼好的結果嗎？」

```
做法：
  1. 計算真實策略的績效（比如 Sharpe = 1.2）
  2. 把報酬序列隨機打亂 1000 次，每次都算一個 Sharpe
  3. 看看 1000 次隨機結果中有幾次 Sharpe ≥ 1.2
  
p 值 = 隨機結果 ≥ 真實結果的比例
p < 0.10 → 只有不到 10% 的機率是隨機產生的 → 訊號是真的
```

### 樣本外測試（OOS Holdout）

保留最近一段資料不讓策略「看到」，在這段資料上測試。

```
訓練期：2017-01-01 ~ 2024-01-01（策略用這段資料學習）
測試期：2024-01-01 ~ 2025-06-30（策略「沒見過」的資料）

如果策略在測試期也賺錢 → 比較可靠
如果策略在訓練期很好、測試期很差 → 過擬合
```

> **系統中的門檻：OOS Sharpe ≥ 0.3**

### 構建敏感度（Construction Sensitivity）

換一種投組建構方法，結果是否還好？

```
做法：
  方法 1: 等權配置 → Sharpe 1.0
  方法 2: 訊號加權 → Sharpe 0.8
  方法 3: 風險平價 → Sharpe 0.9

  如果三種方法都不錯 → 結果穩健
  如果只有等權好，其他很差 → 你可能只是剛好選到最有利的方法
```

## 回測引擎

### 再平衡（Rebalance）

定期調整投組，讓實際持倉回到「目標權重」。

```
頻率選項：
  daily  → 每個交易日都調整
  weekly → 每週調整一次
  monthly → 每月調整一次
```

調整越頻繁，投組越貼近目標，但交易成本越高。

### 先賣後買

系統在調整投組時，先處理「要賣的單」，再處理「要買的單」。

為什麼？因為如果你要把 A 股換成 B 股，先買 B 再賣 A 的話，可能會因為帳戶裡沒有多餘的現金而導致 B 的買單被拒絕。先賣 A 釋放現金，再買 B 就沒問題。

### 交割天數（Settlement, T+N）

交易完成後，錢和股票多久才真正到帳。

```
T+0 = 當天到帳（模擬用，不真實）
T+2 = 兩個交易日後到帳（台股標準）
```

### Kill Switch（熔斷機制）

單日虧損超過設定的閾值時，系統自動賣掉所有持股。

```
如果 今天虧損 > 閾值（預設 5%）：
  → 強制賣出所有持倉
  → 暫停交易
  → 冷卻期過後才恢復
```

## 風控規則

每一筆訂單在執行前，都要通過風控引擎的審核。

| 規則 | 門檻 | 白話 |
|------|------|------|
| 個股上限 | 單一股票 ≤ 5% NAV | 不要把雞蛋放在同一個籃子裡 |
| 單筆下單上限 | 單筆 ≤ 2% NAV | 防止手滑下太大的單 |
| 日回撤限制 | 當日虧損 ≤ 3% | 虧太多就停手，明天再來 |
| 胖手指檢查 | 委託價 ±5% 市價以內 | 防止打錯價格 |
| 成交量限制 | 不超過當日成交量 10% | 避免你自己的買賣影響市場 |
| 槓桿限制 | 總曝險 ≤ 100% NAV | 不借錢投資 |

風控的三種決策：

- **APPROVE** → 通過，正常執行
- **REJECT** → 拒絕，不執行
- **MODIFY** → 修改數量後執行（例如下 600 股但超過上限，自動砍到 500 股）
