Part 1 — NNX：Flax 的神經網路 API¶

為何需要 Flax？¶

  • JAX 提供底層工具 — jit、grad、vmap、NumPy 風格的張量運算
  • JAX 缺少抽象層
    • 只用 JAX 寫神經網路的話要處理很多雜事
    • 參數樹、RNG key、BatchNorm running stats、optimizer state 都得自行串接管理
  • Flax = 把雜事整合好的官方框架
    • 工業：Gemma 開源、Gemini / Imagen / Veo 內部應用
    • 生態：HuggingFace Transformers、Scenic、Big Vision 都建立在 Flax 之上

從 nn 到 NNX: API 演進¶

  • v1 flax.nn — 早期實驗，已淘汰
  • v2 flax.linen (2020) — 功能完整，但寫起來不像 Python
    • 建模型分 init / apply 兩步，不像 PyTorch 一行建好
    • Dropout / BatchNorm 內部狀態要自己接進接出
    • 沒辦法對子模組直接呼叫方法
  • v3 flax.nnx (2024) — Module 就是普通 Python 物件
    • 參數即 屬性，可以印、修改或共享
    • 寫起來約等於 PyTorch

Linen 還在，但官方文件已預設使用 NNX。

NNX 的設計原則¶

  • Pythonic — 普通 class，屬性 可以改
    • self.fc1 = nnx.Linear(...) 跟 PyTorch 一樣
  • Explicit — 維度、隨機數都要指定
    • rngs = nnx.Rngs(0) 統一管理數值
  • Eager — model = MyModel(rngs=...) 直接取得可運行的模型
    • 舊版 Linen 要分 init / apply 兩階段
  • 直接呼叫 — y = model(x) 就跑 forward
    • 不需要 model.apply({'params': ...}, x)

寫法是 PyTorch, 實際運行是 JAX

核心物件: Module、Param、Rngs¶

寫法和 PyTorch 幾乎一樣，三個物件講完就能開始寫:

  • nnx.Module — 神經網路單位
    • 繼承 nnx.Module、forward 寫在 __call__、建構時把 rngs 傳進去
    • 建好直接呼叫 y = model(x) — 沒有 init / apply 兩階段
  • nnx.Param — 「這個 array 是要訓練的」標籤
    • 貼上之後: 自動算梯度、被 optimizer 更新、進 checkpoint
    • 對應 PyTorch 的 nn.Parameter; 取值寫 self.W[...]
  • nnx.Rngs — 隨機數派發器
    • JAX 沒有全域隨機數狀態，隨機數要顯式取得
    • rngs.params() 給參數初始化、rngs.dropout() 給 dropout
    • 同 seed 結果可重現, 方便 debug

內建層¶

跟 PyTorch 類似，涵蓋常見的 Layer：

  • 全連接 / 卷積 — nnx.Linear、nnx.Conv、nnx.ConvTranspose
  • 正規化 / 正則化 — nnx.BatchNorm、nnx.LayerNorm、nnx.Dropout
  • 序列 / 注意力 — nnx.LSTMCell、nnx.GRUCell、nnx.MultiHeadAttention、nnx.Embed
  • 激活函數 — nnx.relu、nnx.gelu、nnx.silu、nnx.softmax、nnx.tanh

沒覆蓋到的，可以直接把 jax.numpy 運算寫進 __call__。

In [3]:
class MLP(nnx.Module):
    def __init__(self, din, dhid, dout, *, rngs):
        self.fc1 = nnx.Linear(din, dhid, rngs=rngs)
        self.bn = nnx.BatchNorm(dhid, rngs=rngs)
        self.drop = nnx.Dropout(0.2, rngs=rngs)
        self.fc2 = nnx.Linear(dhid, dout, rngs=rngs)

    def __call__(self, x):
        x = self.fc1(x)
        x = self.bn(x)
        x = nnx.gelu(x)
        x = self.drop(x)
        return self.fc2(x)


model = MLP(784, 256, 10, rngs=nnx.Rngs(0))
y = model(jnp.ones((1, 784)))
print("MLP output shape:", y.shape)

MLP output shape: (1, 10)

Module 狀態: self.x[...] += 1¶

  • Module 內部狀態可以直接改
    • 如 self.x[...] += 1
  • 自訂狀態類別: 繼承 nnx.Variable
    • 如 Param、BatchStat、Count…
  • 透過標籤可依需求操作，如:
    • 只對 Param 求梯度
    • 只存 BatchStat 進 checkpoint
    • 凍結某類參數
In [4]:
class Count(nnx.Variable):
    """Forward counter."""


class CountingLinear(nnx.Module):
    def __init__(self, din, dout, *, rngs):
        self.w = nnx.Param(
            jax.random.normal(rngs.params(), (din, dout))
        )
        self.count = Count(jnp.array(0))

    def __call__(self, x):
        self.count[...] += 1       # in-place update
        return x @ self.w[...]


model = CountingLinear(4, 2, rngs=nnx.Rngs(0))
_ = model(jnp.ones((1, 4)))
_ = model(jnp.ones((1, 4)))
_ = model(jnp.ones((1, 4)))
print(f"forward calls: {int(model.count[...])}")

forward calls: 3

函式式 API: split / merge / update¶

  • 為什麼需要函式式 API？
    • JAX 使用「純函式 + PyTree」機制
    • Module 是可變物件，JAX 無法處理
  • 工具組:
    • split → 把 Module 拆成 (架構, 數值)
    • merge → 用 (架構, 數值) 重建 Module
    • update → 把新數值寫回原 Module
  • 什麼時候會用到？
    • 存 checkpoint（Orbax 只認 PyTree，要先 split）
    • 模型匯出 / 跨環境部署（架構跟值要分開存）
    • 混搭原生 jax.scan / jax.pmap（純 JAX 轉換看不懂 Module）

Part 2 — 訓練流程¶

NNX Transforms¶

  • @nnx.jit — 用法跟 @jax.jit 一樣
    • 純 JAX 無法處理 Module，不能直接 @jax.jit，要自己 split / merge
    • 額外效果: 函式內對 self.x 的修改會傳回外面
  • nnx.value_and_grad — 一次拿到 loss + 梯度
    • 寫法: loss, grads = nnx.value_and_grad(loss_fn)(model)
    • grads 結構自動對應 model 裡所有 nnx.Param
  • nnx.Optimizer(model, tx, wrt=nnx.Param)
    • wrt=nnx.Param 在 Flax 0.11+ 必填, 指定要更新哪一類 Variable
    • 想做 LoRA？把 wrt 換成 LoraParam 即可, 其他自動凍結

訓練迴圈¶

train_step 函式處理:

  1. loss_fn(m) — 跑 forward、回傳 scalar loss
  2. loss, grads = nnx.value_and_grad(loss_fn)(model)
  3. optimizer.update(model, grads) — 更新權重
  4. return loss — 給外層追蹤訓練曲線用

相對於 PyTorch 的簡化: 不需要 zero_grad 和分開的 backward / step。 外面加 @nnx.jit 就能使用 GPU。

Demo: 完整的 train_step¶

In [6]:
import optax

# Toy 4-input -> 2-class classifier
toy_model = MyModel(4, 2, rngs=nnx.Rngs(0))
optimizer = nnx.Optimizer(
    toy_model, optax.adam(1e-3), wrt=nnx.Param  # wrt arg required in Flax 0.11+
)


@nnx.jit
def train_step(model, optimizer, batch):
    def loss_fn(m):
        logits = m(batch["x"])
        return optax.softmax_cross_entropy_with_integer_labels(
            logits, batch["y"]
        ).mean()

    loss, grads = nnx.value_and_grad(loss_fn)(model)
    optimizer.update(model, grads)
    return loss


# Random toy batch
key = jax.random.key(42)
toy_batch = {
    "x": jax.random.normal(key, (32, 4)),
    "y": jax.random.randint(key, (32,), 0, 2),
}

for step in range(5):
    loss = train_step(toy_model, optimizer, toy_batch)
    print(f"step {step}: loss = {float(loss):.4f}")

step 0: loss = 1.0290
step 1: loss = 1.0277
step 2: loss = 1.0265
step 3: loss = 1.0252
step 4: loss = 1.0240

跑一次 train_step 會自動更新:

  • BatchNorm 的 running mean / variance
  • Dropout 內部的 RNG 狀態
  • Optimizer 的 step 計數、Adam 動量、lr schedule…

做法: NNX Module 是可變物件, 直接改狀態就好。Optimizer 也持有 model 的引用, 不用每步手動接 state。

對照 PyTorch¶

NNX 相比於 PyTorch 的簡化: 不需要 zero_grad（不累積梯度）、不需要分開的 backward + step。

步驟 PyTorch Flax NNX
前向 logits = model(x) logits = model(x)
損失 loss = criterion(logits, y) 寫在 loss_fn(model) 內回傳
反向 loss.backward() nnx.value_and_grad(loss_fn)(model)
清梯度 optimizer.zero_grad() 不需要（不累積）
更新 optimizer.step() optimizer.update(grads)
編譯 torch.compile(model) @nnx.jit

Train vs Eval 模式¶

為什麼要切換模式？ 這兩種層在訓練 / 推論行為不一樣:

  • Dropout — 訓練時隨機丟棄部分神經元、推論時全部保留
  • BatchNorm — 訓練用 batch 統計、推論用累積的

NNX 的切換方法:

  • model.train() / model.eval() — 跟 PyTorch 一樣，一次切整個模型
  • 或在 __call__ 顯式傳 deterministic=True/False

警告: 如果沒切 eval，Dropout 還在隨機丟棄，預測結果會無法重現。

FILES = {
    "train_x": "train-images-idx3-ubyte.gz",
    "train_y": "train-labels-idx1-ubyte.gz",
    "test_x":  "t10k-images-idx3-ubyte.gz",
    "test_y":  "t10k-labels-idx1-ubyte.gz",
}


def download_mnist():
    DATA_DIR.mkdir(parents=True, exist_ok=True)
    for fname in FILES.values():
        dst = DATA_DIR / fname
        if not dst.exists():
            print(f"  downloading {fname} ...")
            urllib.request.urlretrieve(MIRROR + fname, dst)


def load_images(path):
    with gzip.open(path, "rb") as f:
        magic, n, rows, cols = struct.unpack(">IIII", f.read(16))
        return np.frombuffer(f.read(), dtype=np.uint8).reshape(n, rows, cols)


def load_labels(path):
    with gzip.open(path, "rb") as f:
        magic, n = struct.unpack(">II", f.read(8))
        return np.frombuffer(f.read(), dtype=np.uint8)


download_mnist()

Ytr = load_labels(DATA_DIR / FILES["train_y"]).astype(np.int32)

Yte = load_labels(DATA_DIR / FILES["test_y"]).astype(np.int32)

print(f"Train: {Xtr.shape}, labels: {Ytr.shape}")

Step 2: CNN 架構¶

卷積:

  • Filter — 一個小窗滑過影像，抓局部 pattern
  • Feature map — 每個 filter 對輸入的響應圖
  • Pooling — 降採樣，給模型平移容忍度

示範用小型 CNN (28×28 → 10 個分類):

  • Conv 1→32 (3×3) → ReLU → AvgPool (28→14)
  • Conv 32→64 (3×3) → BN → ReLU → AvgPool (14→7)
  • Flatten → Linear → Dropout → ReLU → Linear → 10 類
In [8]:
class CNN(nnx.Module):
    def __init__(self, *, rngs):
        self.conv1 = nnx.Conv(1, 32, kernel_size=(3, 3), rngs=rngs)
        self.conv2 = nnx.Conv(32, 64, kernel_size=(3, 3), rngs=rngs)
        self.bn = nnx.BatchNorm(64, rngs=rngs)
        self.drop = nnx.Dropout(0.25, rngs=rngs)
        self.fc1 = nnx.Linear(7 * 7 * 64, 256, rngs=rngs)
        self.fc2 = nnx.Linear(256, 10, rngs=rngs)

    def __call__(self, x):
        x = nnx.avg_pool(nnx.relu(self.conv1(x)), (2, 2), strides=(2, 2))
        x = nnx.avg_pool(nnx.relu(self.bn(self.conv2(x))), (2, 2), strides=(2, 2))
        x = x.reshape(x.shape[0], -1)
        x = self.drop(nnx.relu(self.fc1(x)))
        return self.fc2(x)


model = CNN(rngs=nnx.Rngs(0))
# Sanity check on a single batch
out = model(jnp.asarray(Xtr[:4]))
print("CNN output shape (batch=4):", out.shape)

Step 3: 訓練設定與訓練¶

  • Optimizer — optax.adamw(lr=5e-3) + cosine LR 衰減
    • cosine decay: 學習率從 5e-3 沿餘弦曲線降到接近 0, 收尾更穩
  • Loss — softmax cross-entropy (label 整數)
  • 訓練: 3,000 步, batch 32
  • 評估: 每 500 步看 train / test accuracy
In [9]:
import time

LR, N_STEPS, BATCH_SIZE = 5e-3, 3000, 32
schedule = optax.cosine_decay_schedule(init_value=LR, decay_steps=N_STEPS, alpha=0.02)

model = CNN(rngs=nnx.Rngs(0))
opt = nnx.Optimizer(model, optax.adamw(schedule, b1=0.9), wrt=nnx.Param)


def loss_fn(model, x, y):
    logits = model(x)
    return optax.softmax_cross_entropy_with_integer_labels(logits, y).mean()


@nnx.jit
def cnn_train_step(model, optimizer, x, y):
    loss, grads = nnx.value_and_grad(loss_fn)(model, x, y)
    optimizer.update(model, grads)
    return loss


@nnx.jit
def cnn_eval_step(model, x, y):
    preds = jnp.argmax(model(x), axis=-1)
    return (preds == y).mean()


rng = np.random.default_rng(0)
t0 = time.time()
for step in range(1, N_STEPS + 1):
    idx = rng.integers(0, len(Xtr), size=BATCH_SIZE)
    x, y = jnp.asarray(Xtr[idx]), jnp.asarray(Ytr[idx])
    loss = cnn_train_step(model, opt, x, y)
    if step == 1 or step % 500 == 0:
        model.eval()
        acc = float(cnn_eval_step(model, jnp.asarray(Xte[:2000]), jnp.asarray(Yte[:2000])))
        model.train()
        print(f"  step {step:4d}  loss={float(loss):.4f}  test_acc={acc:.4f}  ({time.time()-t0:.1f}s)")

# Final full-test-set evaluation
model.eval()
preds = []
for i in range(0, len(Xte), 512):
    preds.append(np.array(jnp.argmax(model(jnp.asarray(Xte[i:i+512])), axis=-1)))
preds = np.concatenate(preds)
test_acc = float((preds == Yte).mean())
print(f"\nFinal test accuracy:  {test_acc:.4f}  (wall time {time.time()-t0:.1f}s)")

核心概念對照: PyTorch ↔ Flax NNX¶

概念 PyTorch Flax NNX
模組定義 class M(nn.Module) class M(nnx.Module)
參數宣告 nn.Parameter(tensor) nnx.Param(jax_array)
層 nn.Linear(in, out) nnx.Linear(in, out, rngs=rngs)
Forward def forward(self, x) def __call__(self, x)
建模型 model = M() model = M(rngs=nnx.Rngs(0))
拿參數 model.parameters() nnx.state(model, nnx.Param)
反向 loss.backward() nnx.value_and_grad(loss_fn)(model)
更新 optimizer.step() optimizer.update(grads)
JIT torch.compile(model) @nnx.jit
存檔 torch.save(state_dict) nnx.split(model) → Orbax

Part 4 — Bonus: 用 NNX 做 LoRA 微調¶

LoRA 是什麼¶

  • 大模型微調的常見技巧
    • 原版微調訓練所有參數，慢且耗記憶體
    • LoRA 凍結原權重，只訓練兩個低秩小矩陣
  • 數學: 原本 $y = Wx$，改成 $y = Wx + BAx$
    • $A$、$B$ 是低秩矩陣，可訓練參數量降到 ~1%

為什麼 NNX 特別適合¶

  • Module 是可變物件 → 直接替換屬性, 不用重建模型
    • model.linear = LoraLinear(model.linear, rank=4) 一行
  • 給 A、B 新標籤: class LoraParam(nnx.Variable)
  • wrt=LoraParam 只挑新標籤訓練 — 原權重自動凍結

同樣的操作在 PyTorch 通常需要寫不少 boilerplate

class LoraParam(nnx.Variable):
    """Trainable, but separable from the base nnx.Param."""


class LoraLinear(nnx.Module):
    def __init__(self, base: nnx.Linear, rank: int, *, rngs):
        self.base = base                       # frozen
        din, dout = base.in_features, base.out_features
        self.A = LoraParam(jax.random.normal(rngs.params(), (din, rank)) * 0.01)
        self.B = LoraParam(jnp.zeros((rank, dout)))

    def __call__(self, x):
        return self.base(x) + (x @ self.A[...]) @ self.B[...]


# Surgery — swap one attribute on the live model
base_model = MyModel(4, 2, rngs=nnx.Rngs(0))
base_model.linear = LoraLinear(base_model.linear, rank=2, rngs=nnx.Rngs(1))

y = base_model(jnp.ones((1, 4)))
print("LoRA-wrapped model output:", y)
