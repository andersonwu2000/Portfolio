# strategies — 策略邏輯

回測程式所執行的策略。核心是「**營收動能**選股」——以兩道普適篩網界定可投資範圍，再用營收動能訊號選股。分層設計、各司其職。

## 主策略一頁懂

```
每月第一個交易日：

  1. 可投資範圍（與因子無關的普適篩網）：
     - 足夠的成交量（買得進、賣得出）
     - 收盤價 ≥ NT$10（濾掉雞蛋水餃股的微結構雜訊）
     - 有足夠的營收歷史（含 15 天公布延遲，防未來函數）

  2. 營收動能訊號（策略核心）：
     - 只看營收在加速的股票（近 3 個月平均 > 近 12 個月平均）
     - 要求創新高連續至少兩個月（濾掉一次性大訂單的雜訊、並降低換手）
     - 依動能指標（以創新高為主）由強到弱排序，取前 N 名
       （創新高過大的近零基期假象會被封頂）

  3. 配置——等權：各檔等權、單檔設上限、保留少量現金。
     （訊號以 rank-IC 驗證 → 只證明排序有效，故等權而非依基數加權；
      亦避免少數近零基期極端值主導配權、徒增單一個股風險。）

  4. 大盤過濾：偵測大盤為空頭或盤整時，整體持倉降到三成。

  5. 不輕易換股：現有部位偏離目標不大時就不動，降低交易成本
     （賣出成本高，故賣出的容忍門檻設得比買進更寬）。
```

一個月內只隨市值波動更新淨值，不交易。

> 早期版本還加了價格趨勢／動能篩網（站上 60 日均線、近 60 日報酬 > 0）與年增 ≥ 10% 門檻；
> 逐項 ablation 顯示價格篩網對「營收動能」主張是混淆、且幾乎不貢獻 Sharpe，年增門檻則扣分——三者皆已移除（見 `Analysis/`）。

## 各檔案角色

| 檔案 | 角色 |
|------|------|
| `revenue_momentum.py` | 主策略：篩選 + 排序 + 配置 + 不輕易換股 |
| `revenue_momentum_hedged.py` | 在主策略外加一層大盤過濾（空頭/盤整縮倉） |
| `momentum.py` | 教科書 12-1 價格動量，作為對照基準 |
| `base.py` / `optimizer.py` / `registry.py` | 策略基底、權重配置、名稱對照等基礎建設 |

> 為什麼這樣設計、各步驟的取捨與效果，見 `Analysis/` 的研究分析。
