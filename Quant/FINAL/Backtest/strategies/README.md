# strategies — 策略邏輯

回測程式所執行的策略。核心是「營收動能 + 價格趨勢確認」的選股法。

## 主策略一頁懂

```
每月第一個交易日：

  1. 篩選候選池（同時滿足）：
     - 有足夠的營收歷史（含 40 天公布延遲）
     - 營收年增率達門檻
     - 近 3 個月平均營收 > 近 12 個月平均（營收在加速）
     - 收盤價 > 60 日均線、近 60 日報酬 > 0（價格趨勢向上）
     - 足夠的成交量（買得進、賣得出）

  2. 排序：依「營收動能」指標由強到弱，取前 N 名。
     （動能指標可用「加速度」「創新高」或兩者集成；研究發現分散持有 + 創新高的組合最佳。）

  3. 配置：依訊號強弱加權，單檔設上限，保留少量現金。

  4. 大盤過濾：偵測大盤為空頭或盤整時，整體持倉降到三成。

  5. 不輕易換股：現有部位偏離目標不大時就不動，降低交易成本
     （賣出成本高，故賣出的容忍門檻設得比買進更寬）。
```

一個月內只隨市值波動更新淨值，不交易。

## 各檔案角色

| 檔案 | 角色 |
|------|------|
| `revenue_momentum.py` | 主策略：篩選 + 排序 + 配置 + 不輕易換股 |
| `revenue_momentum_hedged.py` | 在主策略外加一層大盤過濾（空頭/盤整縮倉） |
| `momentum.py` | 教科書 12-1 價格動量，作為對照基準 |
| `base.py` / `optimizer.py` / `registry.py` | 策略基底、權重配置、名稱對照等基礎建設 |

> 為什麼這樣設計、各步驟的取捨與效果，見 `Analysis/` 的研究分析。
