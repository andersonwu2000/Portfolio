# Backtest — 回測程式與策略

我們自建的回測程式與策略邏輯。它的角色是：在嚴謹、貼近真實的條件下，重現「若照這個策略操作，歷史上會發生什麼」，並讓策略與各種對照組能被公平地比較。

## 回測程式的設計重點

- **逐時點重算可投資範圍**：每個換股日，只考慮當下真的買得到的股票（有成交、上市夠久、具流動性），並**保留已下市公司**——避免「只看活下來的贏家」而高估績效（倖存者偏差）。
- **嚴格防未來函數**：策略在任一時點只能看到當時已公開的資訊。台股月營收於次月才公布，故延後 40 天才允許使用；若提早採用，便構成未來函數（look-ahead），使回測失真。
- **貼近真實的台股成本**：買、賣成本不對稱（賣出含證交稅，約為買進的三倍），如實扣減。
- **公平比較**：策略與所有對照組（隨機選股、被動大型股、ETF）共用**同一套程式、同一動態股池**，差距才有意義。

## 策略邏輯

每月挑選「營收動能強、且價格趨勢向上」的股票，分散持有、單檔設上限，並在大盤轉弱時降低整體持倉。完整的篩選與排序邏輯見 `strategies/` 的說明。

## 未模擬的執行細節

為求方法透明、依賴最少，回測程式並未模擬部分純執行面的細節（例如漲跌停拒單、整股單位、結算延遲、大單滑價衝擊）。這些會使絕對數字略為樂觀，但由於**所有對照組皆採用同一套簡化的程式，策略與基準之間的相對高下不受影響**。
