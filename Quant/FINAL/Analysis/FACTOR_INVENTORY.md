## 4. 核心教訓（g 視角，全新窗）
1. **條件 ICIR ≠ g-alpha（再次釘死，且更乾淨）**：條件 ICIR 最高那批（divyield +0.48 / ep +0.40 / roe_stab +0.41 / st_rev / trustcon / lowvol，**全 t>3**）**全是 g-扣分項**——它們是防禦/價值/低波，有預測力，但把組合從高-g 動能贏家拉向低報酬安全股。ICIR 高 ≠ 加 g。
2. **流動性 ＝ 約束、非參數**（§1）。
3. **g 免疫於「Sharpe 獎勵的犧牲報酬防禦篩網」**（`liq1e8_stab_pool_interaction.py`）：op_margin_stab 當池篩 F50 的 **Sharpe 1.07 > baseline 0.99、MDD 更低**——**舊 Sharpe 目標會「發現」它是改善（如當年低波池 1.8）**；g 正確否決（g 19.1 < 20.1，報酬被犧牲）。**結論：舊那些漂亮的「池×因子交互」有一部分根本是 Sharpe 在共謀，獎勵防禦性篩網。換 g 擋掉一整類假發現。**
4. **池篩（conditional）vs 排序（加性）**：對 op_margin_stab，硬篩越緊 g 越爛（F50 19.1 / F33 14.3 / F25 11.9，全 < 排序混合 22.1）→ **此因子是加性、無額外 conditional 交互**（與舊低波池的「neither-alone-but-combination」相反；該假設類仍有效，只是這個因子不屬於它）。
